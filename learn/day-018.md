L0013:     apples:apples if you change the vocab size. The way this works is that instead of just
L0014:     calculating the average loss as usual, you calculate the sum loss, and independently
L0015:     also the sum bytes (of all the target tokens), and divide. This normalizes the loss by
L0016:     the number of bytes that the target tokens represent.

L0018:     The added complexity is so that:
L0019:     1) All "normal" tokens are normalized by the length of the token in bytes
L0020:     2) No special tokens (e.g. <|bos|>) are included in the metric - they are masked out.
L0021:     3) No actively masked tokens (using ignore_index of e.g. -1) are included in the metric.
