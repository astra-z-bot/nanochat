## Textbook notes
### 1) System context
Week 1 builds operational literacy. The goal is not model theory yet; it is execution confidence. You should be able to answer: which script runs which stage, which file owns each stage's logic, and what artifacts move between stages. Treat this week as system cartography: identify boundaries, interfaces, assumptions, and reproducibility controls.

### 2) Algorithmic lens
At this stage, algorithmic focus is on pipeline interfaces rather than numerical optimization. Document invariants: input schema, output schema, side effects, and failure behavior.

### 3) Day-specific interpretation
For **Week 1 integration notes**, prioritize reading for control-flow and invariants first, then return for optimization details. Your notes should explicitly connect each important behavior to a file and (when possible) a concrete symbol.

## Performance and stability notes
- Environment drift (Python, torch, driver, lock-file mismatch) is the #1 source of false debugging trails.
- Do not tune before you can run baseline scripts reproducibly.
- Always tie observed behavior to exact command + commit + environment snapshot.

## Common failure modes
- Assuming `--help` parity across scripts without reading argument defaults.
- Treating `runs/*.sh` as implementation instead of preset configuration wrappers.
- Skipping data-path shape checks and discovering mismatch late in training.
