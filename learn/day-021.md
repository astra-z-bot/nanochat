Week 3 is optimization and train-loop control. Build a deterministic mental model of: init -> batch -> forward -> loss -> backward -> step -> log -> save. Checkpoint correctness and optimizer-state continuity are mandatory for serious experiments.

### 2) Algorithmic lens
For AdamW-like optimizers, reason through m/v moments, bias correction, and decoupled weight decay. Treat scheduler behavior as part of algorithm definition, not an external decoration.

### 3) Day-specific interpretation

## Performance and stability notes
- Batch shape, accumulation, and distributed sync points dominate runtime behavior.
- Checkpoint cadence is a tradeoff between safety and IO overhead.
- Metric logging frequency can become non-trivial overhead at scale.

## Common failure modes
- Resuming model weights without optimizer/scheduler state.
- Reading loss trends without checking token accounting and data order.
- Changing multiple knobs at once and losing causal attribution.
