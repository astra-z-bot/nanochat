## Focus
Entrypoints in scripts/

## Core material
- Each script corresponds to one pipeline stage: tokenizer, base model, chat finetuning, evaluation, or serving.
- Entrypoints parse stage-specific arguments and hand off work to `nanochat/*` modules.
- Understanding script boundaries is necessary for reproducible experiments and debugging.

## Code pointers
- `scripts/base_train.py` — Base pretraining entrypoint.  \n  Key symbols: `build_model_meta`, `disable_fp8`, `get_scaling_params`, `get_lr_multiplier`, `get_muon_momentum`, `get_weight_decay`
- `scripts/base_eval.py` — Base checkpoint evaluation.  \n  Key symbols: `ModelWrapper`, `load_hf_model`, `get_hf_token_bytes`, `place_eval_bundle`, `evaluate_core`, `main`
- `scripts/chat_sft.py` — Supervised chat finetuning.  \n  Key symbols: `sft_data_generator_bos_bestfit`, `get_lr_multiplier`, `get_muon_momentum`
- `scripts/chat_rl.py` — RL stage for chat model.  \n  Key symbols: `get_batch`, `run_gsm8k_eval`, `get_lr_multiplier`
- `scripts/chat_eval.py` — Chat model evaluation.  \n  Key symbols: `run_generative_eval`, `run_categorical_eval`, `run_chat_eval`
- `scripts/chat_cli.py` — Interactive CLI inference.
- `scripts/chat_web.py` — Web UI serving entrypoint.  \n  Key symbols: `Worker`, `WorkerPool`, `ChatMessage`, `ChatRequest`, `validate_chat_request`
- `scripts/tok_train.py` — Tokenizer training entrypoint.  \n  Key symbols: `main`
- `scripts/tok_eval.py` — Tokenizer evaluation entrypoint.  \n  Key symbols: `main`
